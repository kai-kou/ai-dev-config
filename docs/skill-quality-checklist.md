# Skill 品質監査チェックリスト

新規スキル作成・既存スキル改善時の品質チェック。Anthropic公式ベストプラクティス準拠。

---

## 1. Frontmatter（必須）

- [ ] `name` が kebab-case で64文字以内
- [ ] `description` が What+When+Trigger パターンに準拠
- [ ] `description` が1024文字以内
- [ ] 公式10フィールド以外のカスタムフィールドを追加していない
- [ ] `allowed-tools` が最小権限原則に従っている（設定する場合）

### 公式10フィールド参照

| フィールド | 用途 |
|-----------|------|
| `name` | スキル名（lowercase/numbers/hyphens） |
| `description` | 説明（Claude自動選択の判断に使用） |
| `argument-hint` | オートコンプリート時のヒント |
| `disable-model-invocation` | Claude自動起動の禁止 |
| `user-invocable` | /メニュー表示制御 |
| `allowed-tools` | 許可ツール |
| `model` | 使用モデル |
| `context` | 実行コンテキスト（fork等） |
| `agent` | エージェントタイプ |
| `hooks` | ライフサイクルフック |

---

## 2. Description品質

- [ ] 動詞で始まっている（一人称「I can...」を避ける）
- [ ] What（何をするか）が明確
- [ ] When/Trigger（いつ使うか）が含まれている
- [ ] ユーザー起動型: 日本語トリガー例が「」で囲まれている
- [ ] 内部スキル: 呼び出し元と呼び出し条件が記述されている

---

## 3. 本文構造

### 必須セクション

- [ ] タイトル（h1）が1つだけ存在する
- [ ] 1行の概要文がタイトル直後にある
- [ ] **使用タイミング** または同等のセクションがある
- [ ] **ワークフロー**（Step 1, Step 2...）が記述されている
- [ ] **使用例** が1つ以上ある

### 推奨セクション（SP 3以上）

- [ ] **品質チェックリスト** がある
- [ ] **オプション** テーブルがある（引数を受け取る場合）

### 大規模スキル向け（SP 5以上 / 100行超）

- [ ] **トラブルシューティング** セクションがある
- [ ] **制限事項** が明記されている
- [ ] **前提条件** が記述されている
- [ ] **関連ファイル** テーブルがある

---

## 4. Markdown品質

- [ ] 見出し階層が正しい（h1→h2→h3、飛ばしなし）
- [ ] テーブルの列が揃っている
- [ ] コードブロックに言語指定がある
- [ ] ファイルパス参照先が実在する
- [ ] リンク切れがない

---

## 5. Progressive Disclosure（該当する場合）

- [ ] SKILL.mdのコア部分が150行以内に収まっている
- [ ] 補助ファイルは `references/` に配置されている
- [ ] SKILL.mdから補助ファイルへの相対リンクが有効
- [ ] 補助ファイルが自己完結している（単独で理解可能）

---

## 6. 登録・連携

- [ ] README.md の対応テーブルに追加済み
- [ ] skills-catalog.md に登録済み（Claude Code スキルの場合）
- [ ] 関連するコマンドファイルがある場合、相互参照が正しい
- [ ] `~/.cursor/` への同期を実施（または予定）
